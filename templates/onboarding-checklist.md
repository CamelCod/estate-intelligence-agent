# Onboarding Checklist

Customer: _______________
Property: _______________
Date: _______________

## Step 1 — Camera Audit
- [ ] List all cameras: brand, location, model number
- [ ] Confirm RTSP access: test each stream URL
- [ ] Note any cameras that are offline or inaccessible
- [ ] Flag incompatible cameras
- [ ] Agree on which cameras are in scope for briefings

## Step 2 — Staff Roster Setup
- [ ] Collect all staff names and roles
- [ ] Set expected arrival window per staff member per day
- [ ] Note which days each staff member is expected
- [ ] Confirm which camera covers the main entrance they use

## Step 3 — Owner Preferences
- [ ] WhatsApp number confirmed and verified
- [ ] Preferred briefing time set (default: 8pm)
- [ ] Language preference: English / Arabic / both
- [ ] Notification preference: briefing only / briefing + anomaly alerts
- [ ] Zones of interest flagged (e.g., "always tell me about the garage")

## Step 4 — Test Briefing
- [ ] Generate a test briefing using sample or yesterday's data
- [ ] Send to owner's WhatsApp
- [ ] Confirm receipt and readability
- [ ] Collect feedback: too long / too short / missing anything?

## Step 5 — Go Live
- [ ] First live briefing scheduled
- [ ] Owner knows how to ask follow-up questions (reply to WhatsApp message)
- [ ] Owner has support contact
- [ ] Billing confirmed (trial end date and next charge date noted)
